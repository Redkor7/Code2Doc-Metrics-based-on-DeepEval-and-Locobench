### Описание
Показатель корректности инструмента - это агентский показатель LLM, который оценивает способность вашего агента LLM вызывать функции/инструменты. Он рассчитывается путем сравнения того, действительно ли были вызваны все инструменты, которые, как ожидается, будут использоваться, и был ли выбор инструментов, сделанный агентом LLM, наиболее оптимальным.
### Необходимые аргументы 
Чтобы использовать ToolCorrectnessMetric, вам нужно будет указать следующие аргументы при создании LLMTestCase: 
- input 
- actual_output 
- tools_called`
- expected_tools
### Пример использования:
```python
from deepeval import evaluate  
from deepeval.test_case import LLMTestCase, ToolCall  
from deepeval.metrics import ToolCorrectnessMetric  
  
test_case = LLMTestCase(  
input="What if these shoes don't fit?",  
actual_output="We offer a 30-day full refund at no extra cost.",  
# Replace this with the tools that was actually used by your LLM agent  
tools_called=[ToolCall(name="WebSearch"), ToolCall(name="ToolQuery")],  
expected_tools=[ToolCall(name="WebSearch")],  
)  
metric = ToolCorrectnessMetric()  
  
# To run metric as a standalone  
# metric.measure(test_case)  
# print(metric.score, metric.reason)  
  
evaluate(test_cases=[test_case], metrics=[metric])
```

При создании метрики есть 8 необязательных параметров: 
- available_tools: список вызовов инструментов, который дает представление обо всех инструментах, доступных вашему агенту LLM. Этот список используется для оценки возможностей вашего агента по выбору инструментов. 
- threshold: значение с плавающей запятой, представляющее минимальный порог прохождения, по умолчанию равно 0,5. 
- evaluation_params: список параметров набора инструментов, указывающий на строгость критериев корректности, доступными параметрами являются параметры набора инструментов.ВХОДНЫЕ ПАРАМЕТРЫ и параметры набора инструментов.выход. Например, при вводе списка, содержащего ToolCallParams.INPUT_PARAMETERS, но исключающего ToolCallParams.ПРИ выводе инструмент будет считаться правильным, если имя инструмента и входные параметры совпадают, даже если выходные данные не совпадают. По умолчанию используется пустой список. 
- include_reason: логическое значение, которое, если ему присвоено значение True, будет содержать причину для оценки. По умолчанию установлено значение True. 
- strict_mode: логическое значение, которое, если ему присвоено значение True, устанавливает двоичную оценку по метрике: 1 для совершенства, 0 в противном случае. Оно также переопределяет текущее пороговое значение и устанавливает его равным 1. По умолчанию используется значение False. 
- verbose_mode: логическое значение, которое, если ему присвоено значение True, выводит промежуточные шаги, используемые для вычисления указанного показателя, на консоль, как описано в разделе "Как это вычисляется". По умолчанию установлено значение False. 
-  should_consider_ordering: логическое значение, которое, если ему присвоено значение True, будет учитывать порядок, в котором были вызваны инструменты. Доступно только для параметров TOOLCALL.TOOL и по умолчанию имеет значение False.
- should_exact_match: логическое значение, для которого при установке значения True требуется, чтобы tools_called и expected_tools были точно такими же. Доступно для параметров TOOLCALL.Параметры TOOL и TOOLCALL.INPUT_PARAMETERS и по умолчанию имеет значение False.
### Как происходит рассчет?

Оценка метрики корректности инструмента рассчитывается с помощью следующих шагов: Найдите детерминированную оценку для tools_called с использованием ожидаемых_tools, используя следующее уравнение:

Tool Correctness = Number of Correctly Used Tools (or Correct Input Parameters/Outputs) / Total Number of Tools Called

Этот показатель оценивает точность использования инструментов вашим агентом путем сравнения инструментов_, вызываемых вашим агентом LLM, со списком ожидаемых инструментов. Оценка, равная 1, означает, что все инструменты, используемые вашим агентом LLM, были вызваны правильно в соответствии со списком expected_tools, should_consider_ordering и should_exact_match, в то время как оценка, равная 0, означает, что ни один из инструментов, которые были вызваны, не был вызван правильно.​

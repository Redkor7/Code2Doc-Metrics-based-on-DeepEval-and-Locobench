Для некоторых метрик возможно настроить шаблон промта для их рассчета ответа
### Пример для AnswerRelevancyMetric:

Поскольку AnswerRelevancyMetric от deepeval оценивается LLM-a-a-judge, можно повысить точность своих показателей, переопределив шаблоны запросов deepeval по умолчанию. 

Это особенно полезно, если: используется пользовательский оценочный LLM, особенно для небольших моделей, которые имеют более слабые возможности для выполнения инструкций. Есть желание настроить примеры, используемые в шаблоне AnswerRelevancyTemplate по умолчанию, чтобы они лучше соответствовали ожиданиям.

```python
from deepeval.metrics import AnswerRelevancyMetric  
from deepeval.metrics.answer_relevancy import AnswerRelevancyTemplate  
  
# Define custom template  
class CustomTemplate(AnswerRelevancyTemplate):  
@staticmethod  
def generate_statements(actual_output: str):  
return f"""Given the text, breakdown and generate a list of statements presented.  
  
Example:  
Our new laptop model features a high-resolution Retina display for crystal-clear visuals.  
  
{{  
"statements": [  
"The new laptop model has a high-resolution Retina display."  
]  
}}  
===== END OF EXAMPLE ======  
  
Text:  
{actual_output}  
  
JSON:  
"""  
  
# Inject custom template to metric  
metric = AnswerRelevancyMetric(evaluation_template=CustomTemplate)  
metric.measure(...)
```

1. Импортируем базовый шаблон:

```python
from deepeval.metrics.answer_relevancy import AnswerRelevancyTemplate
```

2. Создаем свой класс:
```python
class CustomTemplate(AnswerRelevancyTemplate):
```

Наследуемся от стандартного шаблона

3. Переопределяем метод generate_statements:
```python
@staticmethod
def generate_statements(actual_output: str):
    return f"""Given the text... JSON:"""
```


Вместо стандартного промпта подставляем свой. Здесь можно:
- Переписать инструкцию: Более понятным языком объяснить модели, что нужно сделать.
- Добавить свой пример (few-shot): В коде показано, как добавить пример разбора предложения про «Retina-дисплей». Это помогает маленьким моделям.
- Изменить формат вывода: Промпт просит вернуть результат в строгом формате JSON.

4. Используем свой шаблон в метрике:
```python
metric = AnswerRelevancyMetric(evaluation_template=CustomTemplate)
```

При создании метрики мы просто передаем наш класс в параметр evaluation_template. Теперь она будет оценивать ответы по вашим правилам.

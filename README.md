# Code2Doc-Metrics-based-on-DeepEval-and-Locobench

Данный проект направлен на изучение различных метрик для измерения качества работы LLM (в частности метрик в deepEval и Locobench), а также на разработку
своих метрик с помощью DeepEval. Основное направление - это метрики для оценки LLM, которые пишут документацию по исходному репозиторию проекта.

## Реализованы следующие метрики: 

### Метрики для LLM, которые пишут документацию по отдельным функциям в проекте
- Semantic Equivalence (Замена BLEU/ROUGE-L)

  Эта метрика сравнивает сгенерированную документацию с эталонной, но оценивает совпадение смысла, а не конкретных слов. Проверяется насколько сгенерированная документация по смыслу совпадает с эталоном. 

  Кратко как работает: Сначала судья достает из эталона набор фактов по типу: "Функция возвращает int". Затем судья смотрит в сгенерированную ИИ дукументацию, берет каждый факт и пытается понять описан ли этот факт в документации именно по смыслу, а не по совпадению слов. Далее вычисляется оценка на основе количества совпадений.
- Code Faithfulness (Отсутствие галлюцинаций)

  Эта метрика оценивает все ли что написала нейронка действительно правда и соответствует функции или она что-то выдумала/не так описала. 

  Кратко как работает: Сначала судья достает из сгенерированной ИИ документации набор фактов по типу: "Функция возвращает int". Затем судья берет каждый такой факт, смотрит в код функции(по которой писалась документация) и пытается понять можно ли этот факт вывесли логически из данного кода или же этот факт является выдуманным нейронкой. Далее вычисляется оценка на основе количества правдивых фактов.
- Documentation Completeness (Информационная полнота)

  Данная метрика проверяет насколько полностью была описана функция нейронкой, не были ли упущены какие-то важные моменты.

  Кратко как работает: Сначала судья делает из кода функции такой чеклист того что должно быть описано. В него входит: описание того для чего эта функция вообще нужна, все параметры что есть у функции, возвращаемое значение, исключения, которые функция может выкинуть в следствии каких-либо ошибок. Затем судья берет этот чеклист и по пунктам смотрит описан ли каждый пункт в сгенерированной ИИ документации или нет. Далее вычисляется оценка на основе количества описанных пунктов.

- Code Coherence (Полезность для разработчика)

  Проверяет насколько легко другому программисту понять замысел функции по документации. Плохая генерация просто построчно переводит код на английский язык, хорошая объясняет намерение. 

  Кратко как работает: Судья берет код функции и документацию сгенерированную ИИ по ней и смотрит насколько хорошо выполнены следующие 3 аспекта: 
  1. Прописана ли цель функции, объясняет ли текст, зачем нужна эта функция в контексте системы?
  2. Избегает ли текст построчного перевода кода?
  3. Написан ли текст на понятном, грамотном техническом английском языке?
  
  Далее вычисляется оценка на основе количества выполненных аспектов.

### Метрики для LLM, которые пишут документацию в общем для кода проекта
- code_text_coherence - метрика, которая определяет насколько связаны между собой блок кода в документации и его описание текстом.
- semantic_completeness - метрика, которая проверяет описала ли нейросеть все важные аспекты проекта в документации или что-то было упущено. Использует "золотой стандарт" (документацию, которая считаеся хорошо соответствующей проекту, написанная или проверенная людьми) при сравнении.
- no_forbidden_word_metric - метрика, которая проверяет есть ли в ответе LLM запрещенное слово, если есть то оценка 0, иначе 1.
- function_parameter_coverage_metric - метрика, которая оценивает качество сгенерированной технической документации, проверяя, чтобы для каждой функции было подробно и чётко объяснено назначение абсолютно всех её параметров.

## Инструкция как запустить тесты в DeepEval:

1. Создать виртуальное окружение Python (если не было создано)
`python -m venv venv`
2. Запустить его
`venv\Scripts\activate`
3. Установить deepEval (если не устновлен)
`install -U deepeval`
4. Установить OPENAI_API_KEY (если не установлен) 
`set OPENAI_API_KEY=...`
5. Установить базовый URL при необходимости(использование proxy) 
 `set OPENAI_BASE_URL=...`
6. написать тест с препиской в файле `test_...`
7. Запустить тест deepeval 
`deepeval test run test_e.py`

P. S. тесты для Code2Doc метрик запускаются командой `python test_C2D_data.py`
